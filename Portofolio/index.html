How I work

My approach

01
Think first, code second
I spend time understanding the problem before writing a single line. The best code is often the code you don't write.
02
Ship small, iterate fast
I believe in tight feedback loops. Small, frequent releases beat long waterfall cycles every time.
03
Performance is a feature
Every millisecond counts. I profile early, optimise with data, and never ship something slow.
04
Tests are documentation
Well-written tests tell the story of your code. I write tests that explain intent, not just behaviour.
05
Collaborate openly
The best ideas come from diverse perspectives. I share work early, give honest feedback, and learn constantly.
06
Own the outcome
I care about the product, not just my part of it. When something breaks at 2am, I'm there fixing it.
